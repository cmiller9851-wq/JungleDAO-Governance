### File: `CONTRIBUTING.md`

1. **Add file** → **Create new file**.  
2. **File name**: `CONTRIBUTING.md`.  
3. **Content**:

   ```markdown
   # Contributing to Jungle DAO

   Thank you for your interest in improving Jungle DAO! Please follow these guidelines to make the contribution process smooth.

   ## How to get started

   1. **Fork** the repository (use the “Fork” button at the top‑right).  
   2. **Clone** your fork locally:

      ```bash
      git clone https://github.com/your-username/jungle-dao.git
      cd jungle-dao
      ```

   3. **Create a new branch** for your work:

      ```bash
      git checkout -b feature/your-feature-name
      ```

   4. Make your changes, then **stage** and **commit**:

      ```bash
      git add <files>
      git commit -m "Brief description of the change"
      ```

   5. **Push** the branch to your fork:

      ```bash
      git push origin feature/your-feature-name
      ```

   6. Open a **Pull Request** against the `main` branch of the upstream repository.

   ## Code style

   - Run the formatter before committing: `npm run format`.  
   - Lint with `npm run lint` and fix any warnings.  
   - Keep commit messages concise and in the imperative mood.

   ## Testing

   - Run unit tests: `npm test`.  
   - Ensure all tests pass and coverage does not drop.

   ## Issues

   - Search existing issues before opening a new one.  
   - Provide a clear description, steps to reproduce, and expected behavior.

   ## License

   By contributing, you agree that your contributions will be licensed under the Apache License 2.0, the same license as the project.
   ```

4. **Commit message**: `Add CONTRIBUTING guide`.  
5. Select **Commit directly to the `main` branch** and tap **Commit new file**.
